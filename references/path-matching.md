# 路径匹配引擎 v2

## 目标
基于用户在特定时间点的自我蒸馏特征（或经由时间线编辑调整后的特征），在抽象人生路径数据库中找到叙事原型最相似的前5-10个案例，为回溯/前向推演提供“社会参照系”。

## 数据库性质（不变）
虚拟概念数据库，存储匿名人生轨迹模式。每个原型包含其特征随时间的变迁函数，以支持不同年龄/人生阶段的匹配。

## 五维匹配算法（不变）
1. 决策模式相似度
2. 时间节律相似度
3. 资源禀赋相似度
4. 叙事主轴相似度
5. 反事实灵活度相似度

综合相似度权重可调，默认 [0.25, 0.15, 0.2, 0.25, 0.15]。

## 时间感知匹配（新增）
当用户回溯到过去时间点并改变选择时，系统需：
- 提取**该时间点的历史特征**（如果用户档案中有时间标签的数据）或根据用户回忆临时赋值。
- 使用当时特征进行匹配，而不是当前特征。这可以找到“当年做出另一种选择的人”的原型，使回溯推演更贴近那个年龄段的典型反应。
- 若用户没有提供足够历史特征，默认降低特征置信度，并使用当前特征进行适度回溯（注明误差可能较大）。

## 匹配过程（扩展）
1. 获取目标时间点的特征向量（当前，或特定回溯时间点）。
2. 执行五维匹配，返回 Top-10 原型。
3. 对于回溯推演，额外筛选那些在相同年龄/人生阶段发生过类似决策改变的原型。
4. 展示时包含原型叙事片段，并强调“这是基于与你当时特征相似者的真实轨迹模式抽象”。

## 匹配结果输出模板

根据你在 [时间点] 的特征，与你最相似的5个原型：

1.原型A (相似度82%)：25岁时离开大公司去初创，经历2年波动后成为技术骨干。

2.原型B (相似度74%)：同样处境下选择留守，后来通过内部转岗寻求突破。
   ...

智能体可选用这些原型作为推演时的历史基线，生成更合理的事件链。

## 应用
匹配原型将继续作为混合引擎中的“个人偏离”计算依据，并且在时间回溯时，可提供“如果当初像原型A那样……”的对比叙事。
