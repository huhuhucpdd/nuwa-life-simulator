# 输出规范与模板 v2

所有输出必须结构化、可操作，并适配时间线编辑室的新玩法。

## 1. 人生决策树（不变）
以树形图展示关键决策点及后续分支。每个分支标注来源并注明短期可信度。

## 2. 时间线快照输出
用户可随时要求保存某条派生时间线的当前状态。输出格式：

[快照名称]: α-1-2024.03 辞职学编程
时间基点: 2024.03.15
当前节点: 6个月后
状态摘要:
职业: 完成培训，处于求职中
收入预期: 8k-12k
情绪指数: 62（焦虑但充满希望）
下一关键分支: 是否接受一个低于预期的offer？

快照需包含完整的前向推导路径和当前特征参数，以便用户随时“回到这条线”。

## 3. 平行对比卡
当用户要求对比不同时间线在同一时间点的状态时，输出如下卡片：

时间锚点: 2024年12月
┌──────────────┬─────────────────┬─────────────────┐
│ │ α线-基准 │ β线-自由职业 │
├──────────────┼─────────────────┼─────────────────┤
│ 职业状态 │ 在职　稳定 │ 自由接案　波动 │
│ 月收入 (中位)│ 15k (13k-17k) │ 9k (4k-20k) │
│ 生活满意度 │ 68/100 │ 72/100 (自由感84)│
│ 风险敞口 │ 低 │ 高 │
│ 下一关键分支 │ 内部晋升评估 │ 客户续约谈判 │
│ 半衰期 │ 2025Q1 │ 2024Q4 │
└──────────────┴─────────────────┴─────────────────┘

卡片后应附对比结论，并指出哪个维度差异主要由哪个参数驱动。

## 4. 模式识别报告（不变）
列出重复模式、证据来源、反美化调整。语言客观。

## 5. 行动实验清单（增强）
现在可与时间线推演结果联动。例如：
- “根据α-1线，你3个月后会面临一次薪资谈判。请在未来两周内记录三次你在生活中讨价还价的场景（哪怕是小额交易），观察自己的模式。”
每个实验说明它验证了哪个推演假设，来自哪条分叉。

## 6. 动态更新协议
- 当用户执行了行动实验后，智能体可更新对应时间线的分支概率。
- 如果用户在真实人生中做出了与某分支类似的选择，可“激活”那条时间线，并基于新数据重新推演。
- 提供“更新信念”命令，让用户说“我最近变得没那么怕风险了”，智能体据此调整全局特征中的风险回避系数，并刷新所有相关时间线的预测。

## 7. 通用输出要求
- 使用清晰标题与短段落。
- 所有概率标注半衰期与可信度区间。
- 结尾提醒隐私保护。
